Introduction to database modelling (in a hurry)

Database modelling is about encoding business truth using tables, constraints, and indexes so your system remains correct under real traffic.

The 20-minute framework

  1. List core entities from product flows (user, order, payment, message, etc.).
  2. Write invariants as DB constraints where possible.
  3. Map top 5 reads + writes and create indexes for those first.
  4. Add history for status transitions and auditing.
  5. Plan migrations before scaling features.
If a business rule matters, encode it in the schema—not only in application code.

Common beginner mistakes

  • Using only surrogate IDs and forgetting UNIQUE constraints for natural keys.
  • Adding indexes without tying them to real query shapes.
  • Keeping only current state and losing lifecycle transitions.
  • Skipping idempotency for payment/order APIs.
