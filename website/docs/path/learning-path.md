# Database modelling learning path

## Beginner

- Entity/relationship basics
- PK/FK/UNIQUE/CHECK constraints
- Normalization and denormalization trade-offs

## Intermediate

- Query-first schema design
- Transaction boundaries + idempotency
- Auditing, soft delete, and retention

## Advanced

- Multi-tenant partitioning and sharding
- Outbox/event-driven consistency
- Materialized read models and backfill strategy

After finishing this path, jump into repository case studies for domain-specific designs.
